## 我完成的工作

### 1. JWT 认证机制

实现了基于 JWT（JSON Web Token）的无状态认证方案，用于微信小程序和 Android 端的 API 认证。

#### 核心功能

1. **Token 生成**：用户登录成功后生成 JWT Token
2. **Token 验证**：每次请求验证 Token 的有效性
3. **Token 刷新**：支持 Token 续期，避免频繁登录
4. **用户信息提取**：从 Token 中解析用户 ID 等信息

#### 技术实现

```java
// JwtUtils.java - Token 工具类
public class JwtUtils {
    // 生成 Token
    public static String generateToken(Long userId, String secret, long expiration)
    
    // 验证 Token
    public static boolean validateToken(String token, String secret)
    
    // 解析 Token
    public static Claims parseToken(String token, String secret)
    
    // 获取用户 ID
    public static Long getUserIdFromToken(String token, String secret)
}
```

#### 配置参数

```yaml
jwt:
  secret: ${JWT_SECRET}
  expiration: 604800  # 7 天（秒）
```

### 2. JWT 拦截器

实现了 JwtInterceptor，对所有需要认证的 API 请求进行拦截和验证。

#### 拦截逻辑

1. **OPTIONS 请求放行**：CORS 预检请求直接通过
2. **Token 提取**：从 Authorization 头提取 Bearer Token
3. **Token 验证**：调用 JwtUtils 验证 Token 有效性
4. **用户信息注入**：将用户 ID 注入到请求属性中
5. **异常处理**：Token 无效或过期时返回 401 错误

#### 白名单配置

以下路径不需要认证：
- `/api/auth/**` - 认证相关接口
- `/api/wx/user/login` - 微信登录
- `/api/wx/studyrooms/**` - 自习室公开接口
- `/api/wx/announcements/**` - 公告公开接口
- `/api/wx/rules/**` - 规则公开接口
- `/api/wx/card/**` - 卡片模块接口
- `/api/card/image/**` - 卡片图片访问
- `/api/test/**` - 测试接口
- `/health/**` - 健康检查接口

### 3. CORS 跨域配置

实现了 WebConfig，配置 CORS 允许跨域请求。

#### 配置内容

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
            .allowedOriginPatterns("*")
            .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
            .allowedHeaders("*")
            .allowCredentials(true)
            .maxAge(3600);
    }
}
```

### 4. 文件上传安全

#### 文件大小限制

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB
```

#### 文件类型验证

- 头像上传：仅允许图片格式（jpg, png, gif）
- 文件存储：使用 UUID 重命名，避免文件名冲突
- 存储路径：基于项目运行目录，避免硬编码绝对路径

### 5. SQL 注入防护

使用 MyBatis 的预编译语句，防止 SQL 注入攻击。

#### 最佳实践

```xml
<!-- 使用 #{} 而非 ${} -->
<select id="findById" resultType="User">
    SELECT * FROM user WHERE id = #{id}
</select>
```

## 遇到的问题和解决

### 1. 问题：Token 过期时间设置
- **描述**：初始设置的 Token 过期时间过短，用户需要频繁登录
- **解决**：将过期时间调整为 7 天（604800 秒），平衡安全性和用户体验

### 2. 问题：CORS 预检请求失败
- **描述**：浏览器发送 OPTIONS 请求时被拦截器拦截，导致跨域失败
- **解决**：在 JwtInterceptor 中对 OPTIONS 请求直接返回 true

### 3. 问题：卡片图片 401 错误
- **描述**：卡片图片路径 `/api/card/image/**` 未加入白名单，导致图片无法访问
- **解决**：在 WebConfig 的 excludePathPatterns 中添加该路径

### 4. 问题：头像上传路径安全问题
- **描述**：使用绝对路径存储头像，不同环境路径不一致
- **解决**：改为基于项目运行目录的相对路径（uploads/avatars/）

### 5. 问题：JWT Secret 硬编码
- **描述**：初始版本将 JWT Secret 硬编码在代码中
- **解决**：改为从环境变量读取，生产环境通过环境变量注入
