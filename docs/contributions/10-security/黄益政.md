## 遇到的问题和解决

1. 问题：Gitleaks扫描时误报较多，将测试代码中的mock数据也识别为密钥
   解决：在`.gitleaks.toml`中配置了allowlist，排除测试目录和mock数据

2. 问题：MobSF需要独立部署服务器，CI中难以直接使用
   解决：配置为可选步骤，使用`continue-on-error: true`，并提供了本地运行的文档说明

3. 问题：部分安全扫描工具（QARK、Mariana Trench）需要Python环境
   解决：在CI中使用pip安装，并设置`continue-on-error: true`避免因工具问题导致CI失败

4. 问题：OWASP Dependency Check需要额外配置Gradle插件
   解决：在`build.gradle.kts`中添加了`org.owasp.dependencycheck`插件

5. 问题：Semgrep默认规则对Android项目覆盖不足
   解决：创建了自定义规则文件`.semgrep.yml`，针对Android特有的安全问题编写规则

通过本次安全审查工作，我深刻认识到移动应用安全的重要性。在开发过程中，开发者往往关注功能实现而忽视安全问题，这可能导致严重的安全漏洞。

本次工作让我学到了：

1. **安全扫描工具的使用**：了解了Gitleaks、Trivy、Semgrep、MobSF等多种安全扫描工具的使用方法和配置方式。

2. **Android安全最佳实践**：学习了OWASP Mobile Top 10安全风险，了解了Android应用常见的安全问题及其防范措施。

3. **CI/CD安全集成**：掌握了如何在GitHub Actions中集成安全扫描，实现自动化的安全检测。

4. **安全配置管理**：学会了如何配置安全扫描规则，平衡误报率和漏报率。

安全是一个持续的过程，不是一次性的工作。通过CI/CD集成安全扫描，可以在代码提交时自动检测安全问题，及时修复漏洞，提高应用的整体安全性。

- 对开发团队进行安全培训，提高安全意识
