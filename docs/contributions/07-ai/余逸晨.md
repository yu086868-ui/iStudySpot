## 我完成的工作

### 1. AI 卡片生成系统

实现了基于 DeepSeek API 的智能卡片生成功能，根据用户学习时长自动生成可收藏的 AI 卡片。

#### 核心功能

1. **稀有度计算**：根据学习时长计算卡片稀有度（N/R/SR/SSR/UR/LR）
2. **主题选择**：从主题库中随机选择卡片主题（励志成长、名人与历史、哲思感悟等）
3. **AI 文本生成**：调用 DeepSeek API 生成卡片 Markdown 内容
4. **AI 图片生成**：基于火山引擎即梦 AI 文生图 3.0 接口生成卡片图片
5. **图片下载与存储**：自动下载生成的图片并存储到服务器本地

#### 架构设计

```
CardController (API 接口)
    ↓
CardServiceImpl (业务逻辑)
    ↓
DeepSeekServiceImpl (AI 调用)
    ↓
CardMapper (数据持久化)
```

#### 接口实现

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/wx/card/generate | POST | 同步生成卡片 |
| /api/wx/card/generate/stream | POST | SSE 流式生成卡片 |
| /api/wx/card/detail | GET | 获取卡片详情 |
| /api/wx/card/list | GET | 获取用户卡片列表 |
| /api/wx/card/image/** | GET | 获取卡片图片 |

#### 稀有度概率分布

### 2. AI 角色对话系统

实现了 AI 角色扮演功能，AI 扮演特定角色（如科学家爱因斯坦）与用户进行对话。

#### 核心功能

1. **角色管理**：支持多个 AI 角色，每个角色有独特的性格和背景
2. **会话管理**：维护用户与角色的多轮对话历史
3. **流式对话**：支持 SSE 流式返回，实现打字机效果
4. **上下文保持**：自动管理对话上下文，保持角色一致性

#### 技术实现

- 使用 DeepSeek API 进行文本生成
- 支持流式响应（Server-Sent Events）
- 会话历史存储在数据库中
- 角色信息从 ai_character 表加载

## 遇到的问题和解决

### 1. 问题：DeepSeek API 超时
- **描述**：生成卡片内容时 API 响应时间过长，导致请求超时
- **解决**：实现 SSE 流式生成，逐步返回内容，避免长时间等待

### 2. 问题：AI 图片生成失败
- **描述**：火山引擎即梦 API 返回错误，图片无法生成
- **解决**：添加重试机制，失败时返回默认图片；优化 prompt 生成逻辑

### 3. 问题：卡片稀有度分布不合理
- **描述**：初始概率设置导致高稀有度卡片过于罕见
- **解决**：调整概率分布，增加学习时长对稀有度的影响权重

### 4. 问题：AI 角色对话上下文丢失
- **描述**：多轮对话时 AI 忘记之前的对话内容
- **解决**：实现会话历史管理，每次请求携带最近 N 条消息作为上下文

### 5. 问题：Agent 工具调用失败
- **描述**：Agent 无法正确解析工具调用参数
- **解决**：优化工具描述和参数定义，添加参数校验和错误处理
