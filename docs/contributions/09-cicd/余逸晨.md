## act 本地验证工作流过程

### 安装 act

```bash
# 使用 scoop 安装（Windows）
scoop install act

# 验证安装
act --version
```

### 运行工作流

```bash
# 运行所有 job
act

# 仅运行 backend job
act -j backend

# 仅运行 frontend job（指定 Node 版本）
act -j frontend -e '{"matrix": {"node-version": "20"}}'

# 使用 verbose 模式查看详细输出
act -v
```

## 遇到的问题和解决

1. 问题：工作流文件路径不正确，GitHub Actions 无法识别
   - 解决：将工作流文件从 `backend/istudyspot-backend/.github/workflows/test.yml` 复制到项目根目录 `.github/workflows/ci.yml`

2. 问题：后端测试需要数据库支持
   - 解决：在工作流中配置了 PostgreSQL 服务容器，提供测试所需的数据库环境

3. 问题：覆盖率徽章显示为 unknown
   - 解决：确保 Codecov Token 正确配置，并在工作流中正确上传覆盖率报告

4. 问题：act 本地运行时缺少环境变量
   - 解决：创建 `.actrc` 文件，设置必要的环境变量和 secrets
