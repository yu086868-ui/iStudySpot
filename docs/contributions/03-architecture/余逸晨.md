### 2. 技术选型

#### 后端框架选择及理由
**选择**：Spring Boot 3.5.11

**理由**：

- 成熟稳定，生态丰富，社区活跃
- 自动配置简化开发，提高效率
- 与 MyBatis、Redis 等组件集成方便
- 内置 Tomcat，部署简单

#### ORM框架选择及理由
**选择**：MyBatis 3.0.5

**理由**：

- SQL 灵活可控，适合复杂查询场景
- 支持动态 SQL，满足不同查询条件
- 性能优秀，轻量级

#### 数据库选择及理由
**选择**：MySQL 8.0

**理由**：
- 事务支持完善，保证数据一致性
- 与 Spring Boot 集成成熟
- 开源免费，社区活跃
- 适合关系型数据存储

#### 缓存选择及理由
**选择**：Redis 6.x

**理由**：
- 高性能内存数据库，适合高并发场景
- 支持分布式锁，解决并发抢座问题
- 数据结构丰富，适合座位状态缓存

#### 认证方案选择及理由
**选择**：JWT

**理由**：
- 无状态认证，适合前后端分离架构
- Token 可携带用户信息，减少数据库查询
- 支持跨域，适合移动端

#### 数据库迁移工具选择及理由
**选择**：Flyway

**理由**：

- 版本化管理数据库脚本
- 团队协作时自动同步数据库结构
- 与 Spring Boot 无缝集成

## 遇到的问题和解决

### 1. 问题：Spring Boot 3.x Redis 配置属性弃用
- **描述**：`spring.redis.host`、`spring.redis.port` 等配置提示弃用
- **解决**：改为使用 `spring.data.redis.host`、`spring.data.redis.port` 新格式

### 2. 问题：MyBatis 字段映射不一致
- **描述**：Java 驼峰命名与数据库下划线命名无法自动映射
- **解决**：在 application.yml 中配置 `map-underscore-to-camel-case: true`
