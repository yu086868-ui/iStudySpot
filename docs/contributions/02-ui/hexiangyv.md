## 遇到的问题和解决

1. 问题：业务流程不清晰，页面之间关系一开始较混乱
   解决：通过“首页 → 选座 → 学习状态”的核心流程进行简化，明确主路径
2. 问题：选座页面视觉表达不直观
   解决：引入颜色区分（绿色=可选，灰色=占用），提升可读性
3. 问题：功能较少，页面显得单薄
   解决：增加卡片式设计与留白，提升界面层次感

------

## 心得体会

在本次 UI 设计过程中，我认识到相比复杂功能，**清晰的业务流程与直观的信息表达更重要**。
通过拆解核心路径（选座 → 学习 → 签退），可以有效降低设计复杂度。同时，统一的设计规范（颜色、间距、字体）对于提升整体体验具有关键作用。
