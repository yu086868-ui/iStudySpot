## 我完成的工作

### 1. Dockerfile 编写

为后端项目编写了多阶段构建的 Dockerfile，优化镜像体积和构建速度。

#### 多阶段构建设计

```dockerfile
# 阶段1: 构建阶段
FROM maven:3.9-eclipse-temurin-17 AS builder
WORKDIR /app
COPY pom.xml .
RUN --mount=type=cache,target=/root/.m2/repository \
    mvn dependency:go-offline -B
COPY src ./src
RUN --mount=type=cache,target=/root/.m2/repository \
    mvn package -DskipTests -B && \
    cp target/*.jar app.jar && \
    rm -rf target/

# 阶段2: 运行阶段
FROM eclipse-temurin:17-jre
WORKDIR /app
COPY --from=builder /app/app.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### 优化点

1. **多阶段构建**：构建阶段使用完整的 Maven + JDK，运行阶段仅使用 JRE，大幅减小镜像体积
2. **依赖缓存**：使用 `--mount=type=cache` 挂载 Maven 仓库缓存，加速依赖下载
3. **离线模式**：先执行 `dependency:go-offline` 下载所有依赖，再复制源码构建
4. **清理构建产物**：构建完成后删除 target 目录，减少中间文件

### 2. docker-compose.yml 配置

编写了两个 docker-compose 文件，分别用于本地开发和项目整体部署。

#### 后端独立 compose 文件

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.4
    container_name: istudyspot-mysql
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_PASSWORD:-123456}
      MYSQL_DATABASE: iseatspace
    healthcheck:
      test: ["CMD", "mysqladmin", "ping"]
  
  redis:
    image: redis:7-alpine
    container_name: istudyspot-redis
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
  
  backend:
    build:
      context: .
      dockerfile: Dockerfile
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
    environment:
      - SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/iseatspace
      - DEEPSEEK_API_KEY=${DEEPSEEK_API_KEY}
```

#### 项目根目录 compose 文件

```yaml
services:
  mysql:
    image: mysql:8.4
    volumes:
      - mysql_data:/var/lib/mysql
      - ./init-db.sql:/docker-entrypoint-initdb.d/01-init.sql
  
  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data
  
  backend:
    build:
      context: ./backend/istudyspot-backend
      dockerfile: Dockerfile
    ports:
      - "8680:8080"
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
  
  admin:
    build:
      context: ./admin
      dockerfile: Dockerfile
    ports:
      - "3001:80"
    depends_on:
      - backend

volumes:
  mysql_data:
  redis_data:
```

#### 配置特点

1. **服务依赖**：使用 `depends_on` 和 `condition: service_healthy` 确保服务启动顺序
2. **健康检查**：为 MySQL、Redis、Backend 配置健康检查
3. **环境变量**：使用 `.env` 文件管理敏感配置（DB_PASSWORD, DEEPSEEK_API_KEY）
4. **数据持久化**：使用 Docker Volume 持久化 MySQL 和 Redis 数据
5. **初始化脚本**：MySQL 支持初始化 SQL 脚本自动执行

- **文档文件**：*.md、docs/
- **测试文件**：e2e-tests/、test_*.js
- **Node 模块**：node_modules/、package.json
- **Docker 配置**：docker-compose.yml（避免循环引用）

### 4. 环境变量管理

使用 `.env` 文件管理敏感配置，不在代码中硬编码。

#### 环境变量列表

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| DB_PASSWORD | MySQL  root 密码 | 123456 |
| DEEPSEEK_API_KEY | DeepSeek API 密钥 | test-key |
| DEEPSEEK_API_URL | DeepSeek API 地址 | https://api.deepseek.com |
| JWT_SECRET | JWT 签名密钥 | - |

#### 使用方式

```bash
# 创建 .env 文件
cat > .env << EOF
DB_PASSWORD=your_password
DEEPSEEK_API_KEY=your_api_key
JWT_SECRET=your_jwt_secret
EOF

# 启动服务
docker-compose up -d
```

### 5. 数据卷管理

使用 Docker Volume 持久化数据，避免容器重启后数据丢失。

#### 数据卷列表

| 卷名 | 挂载路径 | 说明 |
|------|----------|------|
| mysql_data | /var/lib/mysql | MySQL 数据文件 |
| redis_data | /data | Redis 持久化数据 |

#### 管理命令

```bash
# 查看数据卷
docker volume ls

# 备份数据卷
docker run --rm -v istudyspot_mysql_data:/data -v $(pwd):/backup alpine tar czf /backup/mysql_backup.tar.gz /data

# 恢复数据卷
docker run --rm -v istudyspot_mysql_data:/data -v $(pwd):/backup alpine tar xzf /backup/mysql_backup.tar.gz -C /

# 删除数据卷（谨慎使用）
docker volume rm istudyspot_mysql_data
```

## 遇到的问题和解决

### 1. 问题：Maven 依赖下载缓慢
- **描述**：每次构建都需要重新下载依赖，耗时很长
- **解决**：使用 `--mount=type=cache` 挂载 Maven 仓库缓存，依赖只下载一次

### 2. 问题：镜像体积过大
- **描述**：初始镜像包含完整的 JDK 和 Maven，体积超过 1GB
- **解决**：采用多阶段构建，运行阶段仅使用 JRE，镜像体积减小到 200MB 左右

### 3. 问题：MySQL 启动顺序问题
- **描述**：Backend 服务启动时 MySQL 还未就绪，导致连接失败
- **解决**：使用 `depends_on` 配合 `condition: service_healthy`，确保 MySQL 健康后再启动 Backend

### 4. 问题：环境变量未生效
- **描述**：docker-compose 中配置的环境变量未正确传递到容器
- **解决**：使用 `${VAR:-default}` 语法设置默认值，并通过 `.env` 文件管理敏感配置

### 5. 问题：数据持久化问题
- **描述**：容器重启后 MySQL 数据丢失
- **解决**：使用 Docker Volume 持久化数据目录（/var/lib/mysql）

### 6. 问题：构建上下文过大
- **描述**：docker build 时发送的上下文包含大量不必要文件
- **解决**：编写 .dockerignore 排除 target/、.git/、node_modules/ 等目录

## 使用指南

### 快速启动

```bash
# 1. 克隆项目
git clone https://github.com/yu086868-ui/iStudySpot.git
cd iStudySpot

# 2. 配置环境变量
cat > .env << EOF
DB_PASSWORD=your_password
DEEPSEEK_API_KEY=your_api_key
JWT_SECRET=your_jwt_secret
EOF

# 3. 启动所有服务
docker-compose up -d

# 4. 查看日志
docker-compose logs -f backend

# 5. 停止服务
docker-compose down
```

### 访问地址

| 服务 | 地址 | 说明 |
|------|------|------|
| Backend API | http://localhost:8680 | 后端 API 服务 |
| Admin | http://localhost:3001 | 管理后台 |
| MySQL | localhost:3306 | 数据库（需配置密码） |
| Redis | localhost:6379 | 缓存服务 |

### 常用命令

```bash
# 重新构建镜像
docker-compose build --no-cache

# 查看运行状态
docker-compose ps

# 进入容器
docker exec -it istudyspot-backend sh

# 查看资源使用
docker stats

# 清理无用镜像
docker image prune -f
```
