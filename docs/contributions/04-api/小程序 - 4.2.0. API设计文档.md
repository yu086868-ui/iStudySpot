# 小程序端 API 设计文档

> 说明：本文档为 iStudySpot 小程序的完整 API 设计，包含用户认证、自习室管理、座位预约、签到打卡、公告规则等核心功能模块。

---

## 一、API 设计规范

### 1.1 响应格式

所有接口统一返回 JSON 格式：

```json
{
  "code": number,
  "message": "string",
  "data": {},
  "timestamp": number
}
```

**字段说明：**
- `code`: 业务状态码（见状态码说明）
- `message`: 响应消息描述
- `data`: 业务数据，成功时返回具体数据，失败时可为 null
- `timestamp`: 服务器时间戳（毫秒）

---

### 1.2 状态码说明

#### 业务状态码（code）

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或 Token 失效 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 409 | 资源冲突（如重复预约） |
| 422 | 业务逻辑错误（如座位不可用） |
| 500 | 服务器内部错误 |

#### HTTP 状态码

- 200: 成功响应
- 400: 客户端请求错误
- 401: 未授权
- 403: 禁止访问
- 404: 资源未找到
- 500: 服务器错误

---

### 1.3 认证方式

#### Token 认证

需要登录的接口需在请求头中携带 Token：

```
Authorization: Bearer {token}
```

#### Token 有效期

- Access Token: 2小时
- Refresh Token: 7天

---

### 1.4 分页参数

列表接口统一使用以下分页参数：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页数量，默认 20 |

**响应格式：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 20
  }
}
```

- **URL**: `/api/auth/login`
- **方法**: POST

```json
}
```

**响应：**

```json

    "token": "string",
    "refreshToken": "string",

}
```

| 40006 | 预约已过期 |

| 50003 | 签到记录不存在 |
| 50004 | 已经签退 |
| 60001 | 公告不存在 |
| 70001 | 规则不存在 |

### 5.1 接口调用示例

#### 登录示例

```javascript
// 登录
const loginResponse = await fetch('/api/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    username: 'user001',
    password: 'password123'
  })
});

const loginData = await loginResponse.json();
const token = loginData.data.token;

// 使用 Token 调用需要认证的接口
const studyRoomsResponse = await fetch('/api/studyrooms', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
```

---

### 5.2 注意事项

1. **时间格式**: 所有时间字段使用 ISO 8601 格式（如：`2024-01-01T09:00:00Z`）
2. **分页**: 列表接口默认返回 20 条数据，最大不超过 100 条
3. **Token 管理**: Access Token 过期后使用 Refresh Token 刷新
4. **并发控制**: 同一用户同一时间段只能有一个有效预约
5. **签到时效**: 预约开始时间后 15 分钟内可签到，超时自动取消

---

### 5.3 Mock 数据建议

开发阶段建议使用 Mock 数据进行接口模拟，可使用以下工具：
- Mock.js
- JSON Server
- Postman Mock Server
