# 后端说明

后端代码库更适合被描述为一个模块化单体。
控制器、服务、Mapper、SQL 映射与定时任务都运行在同一个 Spring Boot 进程里。

为了让模型贴近真实代码，以下实现细节被明确保留：

- 请求认证是通过拦截器建立请求上下文完成的，而不是通过一个独立部署的认证服务
- AI 与客服对话会话当前保存在服务类的内存结构中
- Agent 工具直接复用现有的自习室、座位、订单与预约规则逻辑，而不是访问一个单独的编排后端
- 生成后的卡牌图片会先写入本地目录，再通过 `CardController` 对外访问
- 统计与公告接口当前仍主要返回占位数据，而不是完整业务实现
- 规则接口大部分来自静态代码或配置，而不是数据库驱动的规则引擎
