# API接口文档

## 1. 认证接口

### 1.1 登录
- **路径**: `/api/auth/login`
- **方法**: `POST`
- **参数**:
  - `username`: 用户名
  - `password`: 密码
- **返回**: 包含token、refreshToken和用户信息的对象

### 1.2 注册
- **路径**: `/api/auth/register`
- **方法**: `POST`
- **参数**:
  - `username`: 用户名
  - `password`: 密码
  - `nickname`: 昵称
  - `phone`: 手机号
  - `studentId`: 学号
- **返回**: 包含userId的对象

### 1.3 刷新令牌
- **路径**: `/api/auth/refresh`
- **方法**: `POST`
- **参数**:
  - `refreshToken`: 刷新令牌
- **返回**: 包含新token和refreshToken的对象

### 1.4 登出
- **路径**: `/api/auth/logout`
- **方法**: `POST`
- **参数**: 无
- **返回**: 成功消息

## 2. 用户接口

### 2.1 获取用户信息
- **路径**: `/api/users/me`
- **方法**: `GET`
- **参数**: 无（从JWT中获取userId）
- **返回**: 用户信息对象

### 2.2 更新用户信息
- **路径**: `/api/users/me`
- **方法**: `PUT`
- **参数**:
  - `nickname`: 昵称
  - `phone`: 手机号
  - `avatar`: 头像URL
- **返回**: 更新后的用户信息对象

### 2.3 修改密码
- **路径**: `/api/users/me/password`
- **方法**: `PUT`
- **参数**:
  - `oldPassword`: 旧密码
  - `newPassword`: 新密码
- **返回**: 成功消息

### 5.2 获取订单列表

## 6. 支付接口

### 6.1 创建支付
- **路径**: `/api/payments`
- **方法**: `POST`
- **参数**:
  - `orderId`: 订单ID
  - `amount`: 支付金额
  - `paymentMethod`: 支付方式
- **返回**: 包含paymentId和paymentUrl的对象

### 6.2 获取支付状态
- **路径**: `/api/payments/{id}`
- **方法**: `GET`
- **参数**:
  - `id`: 支付ID
- **返回**: 支付状态对象

### 6.3 支付回调
- **路径**: `/api/payments/callback`
- **方法**: `POST`
- **参数**:
  - `paymentNo`: 支付编号
  - `success`: 支付是否成功
- **返回**: 回调处理结果

## 12. 响应格式

所有API接口返回的响应格式如下：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {...},  // 具体返回数据
  "timestamp": 1620000000000
}
```

### 错误响应

```json
{
  "code": 400,  // 错误码
  "message": "错误信息",
  "data": null,
  "timestamp": 1620000000000
}
```

## 13. 认证方式

所有需要认证的接口都需要在请求头中携带JWT令牌：

```
Authorization: Bearer <token>
```
