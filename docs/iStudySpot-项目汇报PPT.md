## 第3页：项目背景与目标

### 背景
- 考研、考公、考证人群日益增长，自习室需求旺盛
- 传统自习室管理依赖人工，效率低、易出错
- 高峰期座位争抢、排队等待问题严重

### 目标
- 打造一站式在线自习室预订平台
- 支持多端（微信小程序 + Android + 管理后台）
- 实现智能化管理（AI 对话、AI 卡片生成、自动计费）
- 解决高并发抢座问题（Redis 分布式锁）

### 用户群体
- 学习学生、考研党、考证族、远程办公人群

## 第6页：技术栈

### 后端

- **管理后台**: React + Vite

### DevOps
- **容器化**: Docker + docker-compose
- **CI/CD**: GitHub Actions + Codecov
- **数据库迁移**: Flyway

## 第8页：AI 智能功能

### AI 卡片生成
- 根据学习时长自动生成可收藏的AI卡片
- 6种稀有度：N → R → SR → SSR → UR → LR
- 7大主题类别：励志成长、名人与历史、哲思感悟等
- 支持同步生成和SSE流式生成
- 基于火山引擎即梦AI文生图3.0接口生成卡片图片

### AI 角色对话
- 多角色扮演（客服、历史人物等）
- 支持流式聊天（SSE 打字机效果）
- 会话历史管理，上下文保持

### AI Agent 对话
- 智能工具调用（查询自习室、座位、预约等）
- 多轮对话，自动理解用户意图

## 第11页：微信小程序接口隔离设计

### 双端接口隔离

```
/api/auth/login     →  Android 端
/api/wx/user/login  →  微信小程序端
```

- 微信小程序接口统一前缀 `/api/wx/`
- 独立 Controller 类（Wx*Controller）
- 独立 DTO（WxStudyRoomDTO, WxOrderDTO, WxSeatDTO）
- 与原有接口完全隔离，互不影响

## 第12页：安全与性能

### 安全措施
- **JWT 认证**: 无状态 Token 认证，7天有效期
- **接口拦截**: JwtInterceptor 白名单机制
- **CORS 配置**: 跨域请求控制
- **SQL 注入防护**: MyBatis 预编译语句（#{}）
- **文件上传安全**: 10MB 限制、类型校验、UUID 重命名
- **敏感信息**: 环境变量管理密钥

### 性能优化
- **Redis 分布式锁**: 解决高峰期座位并发抢订
- **Redis 缓存**: 座位状态、价格缓存、热门数据
- **MySQL 索引**: 用户查询、订单时间范围、座位状态
- **Docker 多阶段构建**: 镜像体积优化（1GB → 200MB）
