# iStudySpot 小程序端 API 设计文档（实际版）

> 说明：本文档基于 iStudySpot 后端实际代码编写，与 V5.0 设计文档一一对应，反映实际实现的接口细节。

---

## 一、API 设计规范

### 1.1 响应格式

大部分接口统一返回 JSON 格式：

```json
{
  "code": number,
  "message": "string",
  "data": {},
  "timestamp": number
}
```

**字段说明：**
- `code`: 业务状态码（见状态码说明）
- `message`: 响应消息描述
- `data`: 业务数据，成功时返回具体数据，失败时可为 null
- `timestamp`: 服务器时间戳（毫秒）

> 注意：卡片模块（`/api/wx/card`）使用独立的响应格式 `{"success": boolean, "message": "string", "card": {...}}`

---

### 1.2 状态码说明

#### 业务状态码（code）

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 201 | 创建成功 |
| 400 | 请求参数错误 |
| 401 | 未登录 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 409 | 资源冲突（如重复预约） |
| 422 | 业务逻辑错误（如座位不可用） |
| 500 | 服务器内部错误 |

#### HTTP 状态码

- 200: 成功响应
- 400: 客户端请求错误
- 401: 未授权
- 403: 禁止访问
- 404: 资源未找到
- 500: 服务器错误

---

### 1.3 认证方式

采用微信小程序登录机制，通过 `wx.login()` 获取临时 code，调用后端登录接口完成认证。

后端根据 code 获取 openId，查询用户是否存在，不存在则自动创建用户（自动注册机制）。

登录成功后返回 JWT Token，后续请求在 Header 中携带：

```
Authorization: Bearer <token>
```

Token 有效期：604800 秒（7 天）。

---

### 1.4 分页参数

列表接口统一使用以下分页参数：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页数量，默认 20 |

**响应格式：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 20
  }
}
```

### 2.2 自习室（WxStudyRoom）- 小程序端

```typescript
interface WxStudyRoom {
  id: string;
  name: string;
  description: string;
  location: string;          // 对应数据库 address 字段
  floor: number | null;      // 楼层
  capacity: number | null;   // 容量
  openTime: string;          // HH:mm:ss
  closeTime: string;         // HH:mm:ss
  facilities: string[];      // 设施列表（逗号分隔字符串转为数组）
  image: string;             // 对应数据库 imageUrl 字段
  status: 'open' | 'closed'; // 对应数据库 status 字段(1=open, 0=closed)
  rules: WxRuleDTO[];        // 该自习室的规则列表（仅详情接口返回）
  createdAt: string;
  updatedAt: string;
}
```

---

### 2.3 座位（WxSeat）- 小程序端

```typescript
interface WxSeat {
  id: string;
  studyRoomId: string;       // 对应数据库 roomId 字段
  row: number;               // 对应数据库 rowNum 字段
  col: number;               // 对应数据库 colNum 字段
  seatNumber: string;
  type: 'normal' | 'vip';    // 对应数据库 seatType 字段(1=normal, 2=vip)
  status: string;            // available/booked/occupied/unavailable
  facilities: string[];      // 设施列表（逗号分隔字符串转为数组）
  lastUsedAt: string | null; // 最后使用时间
}
```

### 2.7 规则（Rule）

**规则实体（Rule 实体完整字段，规则模块接口直接返回此实体）：**

```typescript
interface Rule {
  id: number;
  studyRoomId: number | null;    // 关联自习室ID，null 表示通用规则
  category: string;              // booking/checkin/leave/violation/civilized
  title: string;
  content: string;
  priority: number;
  createTime: string;
  updateTime: string;
}
```

**规则 DTO（WxRuleDTO，仅用于自习室详情接口中的 rules 字段）：**

```typescript
interface WxRuleDTO {
  id: string;
  category: string;
  title: string;
  content: string;
  priority: number;
}
```

> 注意：独立规则模块接口（`/api/wx/rules`、`/api/wx/rules/{id}`）返回完整 `Rule` 实体；自习室详情接口中的 `rules` 字段使用精简的 `WxRuleDTO`。

### 2.9 卡片（Card）

```typescript
interface Card {
  uuid: string;
  rarity: string;            // N/R/SR/SSR/UR/LR
  borderTheme: string;
  cardTheme: string;
  themeCategory: string;
  markdown: string;
  imageURL: string;
  createTime: string;        // yyyy-MM-dd HH:mm:ss
  studyDuration: number;     // 分钟

```

#### 稀有度说明

| 稀有度 | 边框主题 | 卡片主题 | 获取概率 |
|--------|----------|----------|----------|
| N | 白/灰 | 普通 | 常见 |
| R | 绿 | 普通 | 较常见 |
| SR | 蓝 | 普通 | 中等 |
| SSR | 紫 | 普通 | 稀有 |
| UR | 金 | 特殊 | 非常稀有 |
| LR | 红 | 特殊 | 极其稀有 |

#### 主题类别

- 励志成长
- 名人与历史
- 哲思感悟
- 自然意象
- 科技未来
- 温柔陪伴
- 隐藏主题（仅 UR/LR 稀有度可获得）

### 5.1 接口调用示例

#### 微信登录示例

```javascript
// 微信登录
wx.login({
  success: async (res) => {

  }
```

### 5.2 注意事项

1. **时间格式**: 预约请求时间支持 ISO 8601（`2024-01-01T09:00:00Z`）或 `yyyy-MM-dd HH:mm:ss` 格式
2. **分页**: 列表接口默认返回 20 条数据
3. **并发控制**: 同一用户同一时间段只能有一个有效预约
4. **卡片认证**: 卡片接口已添加到白名单，无需登录即可访问
5. **卡片图片存储**: 生成的图片存储在服务器本地（默认 `./uploads/cards`），通过 `/api/wx/card/image/` 访问
6. **SSE 超时**: 流式接口超时时间为 120 秒
7. **编码格式**: 所有响应编码为 UTF-8
8. **头像上传**: 使用 `multipart/form-data`，参数名为 `file`，存储路径基于项目运行目录（`uploads/avatars`）
9. **JWT Token**: 有效期 7 天，过期需重新登录
10. **DTO 隔离**: 小程序接口使用独立 DTO（WxStudyRoomDTO/WxOrderDTO/WxSeatDTO），与原有安卓接口互不影响
11. **规则返回差异**: 规则模块接口返回完整 Rule 实体（含 studyRoomId/createTime/updateTime），自习室详情中的 rules 使用精简 WxRuleDTO

## 七、与 V5.0 设计文档的差异说明

| 模块 | V5.0 设计 | 实际实现 |
|------|-----------|----------|
| 用户登录 | `/api/user/login` | `/api/wx/user/login`（路径不同） |
| 登录响应 | 无 token 字段 | 返回 `token` 字段（JWT认证） |
| 自习室数据结构 | `location`/`floor`/`facilities`/`capacity`/`status: open\|closed` | DTO 映射后与 V5.0 一致 |
| 座位数据结构 | `studyRoomId`/`row`/`col`/`type: normal\|vip`/`facilities[]` | DTO 映射后与 V5.0 一致 |
| 预约请求时间 | ISO 8601 | 支持 ISO 8601 和 `yyyy-MM-dd HH:mm:ss` |
| 预约响应 | 完整 Order 字段 | DTO 精简为 `id/userId/studyRoomId/seatId/startTime/endTime/status/checkInTime/checkOutTime/createdAt/updatedAt` |
| 签到记录响应 | 标准分页 | 返回 `list/total/page/pageSize` |
| 规则模块 | `/api/rules` 独立模块 | `/api/wx/rules` 独立模块，返回完整 Rule 实体 |
| 公告模块 | 完整实现 | 已完整实现（非 TODO） |
| 座位布局 | 无 | 新增 `/api/wx/studyrooms/{studyRoomId}/seat-layout` 接口 |
| 卡片模块 | 独立响应格式 | 使用 `{"success", "message", "card"}` 格式，非标准 `{code, data}` |
