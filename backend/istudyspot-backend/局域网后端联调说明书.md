# 局域网分离前后端联调

## 1. 目标

前后端分别运行在不同电脑，通过局域网完成联调。

架构如下：

```text
前端（我的电脑）
↓
浏览器 / fetch / axios
↓
http://后端局域网IP:端口
↓
后端（你的电脑）
```

示例：

```text
前端：
localhost:5173

后端：
192.168.1.107:3000
```

前端直接请求：

```text
http://192.168.1.107:3000
```

无需内网穿透，无需公网，仅在同一局域网内即可。

------

## 2. 后端配置要求

### 2.1 监听地址必须为 0.0.0.0

后端不能仅监听：

```text
127.0.0.1
```

或：

```text
localhost
```

否则仅本机可访问。

需要监听：

```text
0.0.0.0
```

即：

```text
允许局域网设备访问
```

常见示例：

### Node / Express

推荐：

```js
app.listen(3000, "0.0.0.0")
```

或：

```js
app.listen(3000)
```

------

### Flask

```python
app.run(host="0.0.0.0", port=5000)
```

------

### Spring Boot

```
application.yml
server:
  address: 0.0.0.0
  port: 8080
```

------

## 3. 获取局域网 IP

Windows：

CMD 执行：

```cmd
ipconfig
```

查找：

```text
IPv4 Address
```

示例：

```text
192.168.1.107
```

此 IP 即后端访问地址。

------

## 4. 防火墙确认

若浏览器无法访问：

```text
http://IP:端口
```

请检查：

Windows Defender 防火墙

确保：

```text
允许应用访问网络
```

第一次启动服务时若弹出：

```text
允许访问
```

请选择：

```text
允许
```

------

## 5. CORS 跨域配置（推荐提前开启）

前端与后端端口不同：

例如：

```text
前端：
localhost:5173

后端：
192.168.1.107:3000
```

浏览器可能产生跨域限制（CORS）。

常见报错：

```text
Access-Control-Allow-Origin
CORS blocked
```

建议后端开发阶段直接开启 CORS。

### Express

安装：

```bash
npm install cors
```

使用：

```js
const cors = require('cors')

app.use(cors())
```

------

### Flask

```python
from flask_cors import CORS

CORS(app)
```

------

### Spring Boot

控制器：

```java
@CrossOrigin
```

或统一全局配置。

------

## 6. 联调验证

后端启动后，请提供：

```text
局域网IP + 端口
```

例如：

```text
192.168.1.107:3000
```

验证步骤：

浏览器访问：

```text
http://192.168.1.107:3000
```

若可访问或返回接口数据，则说明联调环境正常。

------

## 7. 最终交付内容

后端同学需提供：

```text
1. 后端已启动
2. 监听地址为 0.0.0.0
3. 局域网 IP
4. 端口号
5. 若有跨域问题，已开启 CORS
```

收到后即可进行局域网前后端并行开发与联调。
