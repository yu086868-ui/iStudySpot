# AI角色扮演系统

## 选择与限定功能

选择之前做过的AI角色扮演系统

角色提示词修改为科学家等即可。

项目定义、用户与场景略。

### 核心功能：

1. 与角色对话聊天
2. 多轮对话管理
3. 保持角色一致性
4. 美观的界面与交互辅助

### 非目标：

- 不做知识库问答系统
- 不做复杂Agent（工具调用）
- 不做长期记忆（跨会话）
- 不做语音 / 图片等多模态
- 不做“通用AI助手”



# 核心架构设计

极简性与可用性

## 系统拆分（4模块）

```
Character（角色）
Session（会话）
Prompt Builder（提示词构建）
Context（上下文策略）
```



## 1. Character（角色层）

### 数据结构

```
Character {
  id: string,
  name: string,
  persona: string,
  speaking_style: string
}
```

### 职责

- 提供角色约束（人格 + 表达风格）
- 不参与运行时逻辑

------

## 2. Session（会话层）

### 数据结构

```
Session {
  session_id: string,
  character_id: string,
  messages: Message[]
}

Message {
  role: "user" | "assistant",
  content: string
}
```

### 职责

- 存储对话历史
- 提供上下文输入

------

## 3. Prompt Builder（控制层）

### 模板

```
你正在扮演一个角色，请严格遵守以下设定：

角色名称：{name}
性格：{persona}
说话风格：{style}

要求：
- 始终保持角色语气
- 不要提到自己是AI
- 不要跳出角色

请根据对话继续交流。
```

### 构建函数

```
function buildSystemPrompt(character) {
  return `
你正在扮演一个角色，请严格遵守以下设定：

角色名称：${character.name}
性格：${character.persona}
说话风格：${character.speaking_style}

要求：
- 始终保持角色语气
- 不要提到自己是AI
- 不要跳出角色

请根据对话继续交流。
`;
}
```

------

## 4. Context（上下文策略）

### 策略

```
const MAX_MESSAGES = 10;

function buildContext(messages) {
  return messages.slice(-MAX_MESSAGES);
}
```

### 最终输入结构

```
finalMessages = [
  { role: "system", content: systemPrompt },
  ...recentMessages
];
```

------

## 5. 核心流程（唯一主链路）

```
用户输入
→ 写入 session.messages
→ 构建 systemPrompt
→ 截取 recentMessages
→ 拼接 finalMessages
→ 调用 LLM API
→ 写入 assistant 回复
→ 返回结果
```

------

## 6. 核心函数（后端最小实现）

```
async function chat(session, character, userInput) {
  session.messages.push({
    role: "user",
    content: userInput
  });

  const systemPrompt = buildSystemPrompt(character);

  const recentMessages = session.messages.slice(-10);

  const finalMessages = [
    { role: "system", content: systemPrompt },
    ...recentMessages
  ];

  const response = await callLLM(finalMessages);

  session.messages.push({
    role: "assistant",
    content: response
  });

  return response;
}
```
