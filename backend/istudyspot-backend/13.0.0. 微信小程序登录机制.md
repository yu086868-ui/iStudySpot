如果你的目标是：

* 只支持微信小程序登录
* 不涉及 JWT、Token、RefreshToken
* 不涉及权限系统
* 不涉及手机号绑定
* 先完成最小可运行版本
* 用户头像采用本地存储路径即可

那么建议直接把原来的「用户认证模块」全部删掉，替换为下面这一份。

后续再与你现有文档合并。

---

# 用户模块（User）

## 数据结构

```typescript
interface User {
  id: number;

  // 微信用户唯一标识
  openId: string;

  // 用户昵称
  nickname: string;

  // 用户头像
  avatarUrl: string;

  // 用户状态
  status: 'normal' | 'disabled';

  createdAt: string;
  updatedAt: string;
}
```

---

# 微信登录模块

## 业务说明

小程序启动后：

1. 调用 `wx.login()`
2. 获取临时 code
3. 调用后端登录接口
4. 后端根据 code 获取 openId
5. 查询用户是否存在
6. 不存在则自动创建用户
7. 返回用户信息

采用自动注册机制。

无需账号密码。

---

## 3.1 微信登录

### 接口信息

* URL：`/api/user/login`
* Method：`POST`

---

### 请求参数

```json
{
  "code": "wx_login_code"
}
```

---

### 响应示例（首次登录）

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "isNewUser": true,
    "user": {
      "id": 1,
      "nickname": "微信用户",
      "avatarUrl": "",
      "status": "normal"
    }
  }
}
```

---

### 响应示例（老用户）

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "isNewUser": false,
    "user": {
      "id": 1,
      "nickname": "Theresa",
      "avatarUrl": "/avatar/avatar_1.jpg",
      "status": "normal"
    }
  }
}
```

# 推荐数据库结构

```sql
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    open_id VARCHAR(64) NOT NULL UNIQUE,

    nickname VARCHAR(50) DEFAULT '微信用户',

    avatar_url VARCHAR(255) DEFAULT '',

    status VARCHAR(20) DEFAULT 'normal',

    created_at DATETIME,

    updated_at DATETIME
);
```

---

对于你目前的小程序（预约自习室）来说，其实只需要保留下面 4 个接口即可：

```text
POST   /api/user/login
GET    /api/user/profile
PUT    /api/user/profile
POST   /api/user/avatar
```

这是最小闭环版本。

等后面做：

* 登录态校验
* Token/JWT
* 手机号绑定
* 管理员权限
* 数据同步

再在这个基础上扩展即可，不建议现在一次性设计进去。
