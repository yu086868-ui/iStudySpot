# AI卡片生产系统契约（V1）- to后端

## 1. 模块目标

AI卡片系统作为主学习系统的扩展模块，用于：

> 根据一次有效学习记录，自动生成可收藏的AI卡片。

系统目标不是抽卡玩法本身，而是：

> 通过AI文本、AI图片与稀有度系统，对真实学习投入进行记录与收藏。

卡片系统与主业务相对独立。

主系统仅提供：

```
一次有效学习记录
```

卡片系统负责：

```
生成
保存
展示
收藏
```

## 2. 最终效果（业务目标）

用户完成一次有效学习后：

系统自动：

1 计算稀有度

2 选择主题池

3 生成AI文本

4 生成AI图片

5 保存卡片数据

6 返回卡片信息

7 前端展示卡片并进入收藏系统

业务流程：

```
学习记录
↓

卡片生成请求
↓

服务器生成卡片
↓

AI文本 + AI图片
↓

卡片落库
↓

返回卡片
↓

前端展示/收藏
```



## 3. 系统责任边界

为避免职责混乱，V1采用：

> 前后端责任分离

------

# 前端负责

前端负责：

### 1 用户交互

包括：

- 触发生成
- 加载状态
- 卡片展示
- 收藏页展示

------

### 2 Markdown渲染

负责：

```
Markdown → UI组件
```

包括：

- 标题
- 引用
- 正文
- 图片

------

### 3 视觉与动画

包括：

- 卡面主题
- 边框
- 光效
- 收藏动画

------

### 4 接口调用

调用：

```
/api/card/*
```

不直接调用AI服务。

------

# 后端负责

后端负责：

### 1 稀有度计算

依据：

```
studyDuration
```

计算概率池。

------

### 2 主题分配

依据：

- 稀有度
- 主题池规则

决定：

```
themeCategory
```

------

### 3 AI Prompt组装

后端负责：

- 文本Prompt
- 图片Prompt

前端不拼接AI提示词。

------

### 4 AI接口调用

包括：

- 文本生成
- 图片生成

实现技术不限。

例如：

- 即梦
- OpenAI
- 其他

属于后端内部实现。

------

### 5 数据落库

负责：

- UUID生成
- 数据保存
- 图片URL保存

------

### 6 API返回

返回：

```
Card Data
```

给前端。

------

# 4. 系统数据结构

卡片数据结构：

```
{
  "uuid": "",
  "userID": "",
  "cardID": "",

  "createTime": "",
  "studyDuration": 0,

  "rarity": "",
  "borderTheme": "",
  "cardTheme": "",
  "themeCategory": "",

  "markdown": "",
  "imageURL": ""
}
```

------

# 字段说明

| 字段          | 说明          |
| ------------- | ------------- |
| uuid          | 卡片唯一ID    |
| userID        | 用户ID        |
| cardID        | 模板/生成编号 |
| createTime    | 生成时间      |
| studyDuration | 学习时长      |
| rarity        | 稀有度        |
| borderTheme   | 边框主题      |
| cardTheme     | 卡面主题      |
| themeCategory | 内容主题      |
| markdown      | AI文本        |
| imageURL      | 图片地址      |

------

## 5. API契约

统一：

```
/api/card/*
```

------

### 5.1 生成卡片

接口：

```
POST /api/card/generate
```

作用：

```
根据学习记录生成卡片
```

------

请求：

```
{
  "userID":"123",
  "studyDuration":120
}
```

说明：

| 字段          | 必填 | 说明             |
| ------------- | ---- | ---------------- |
| userID        | 是   | 用户ID           |
| studyDuration | 是   | 学习时长（分钟） |

------

成功返回：

```
{
  "success": true,
  "message":"generate success",
  "card":{
    "uuid":"",
    "rarity":"",
    "borderTheme":"",
    "cardTheme":"",
    "themeCategory":"",
    "markdown":"",
    "imageURL":"",
    "createTime":""
  }
}
```

------

失败返回：

```
{
  "success": false,
  "message":"generate failed"
}
```

错误原因可包括：

- AI生成失败
- 图片失败
- 数据库存储失败
- 参数错误

------

------

### 5.2 获取卡片详情

接口：

```
GET /api/card/detail
```

请求：

```
?id=uuid
```

返回：

```
{
  "success":true,
  "card":{}
}
```

用于：

- 收藏页
- 卡片查看

------

### 5.3 获取用户卡片列表

接口：

```
GET /api/card/list
```

请求：

```
?userID=123
```

返回：

```
{
  "success":true,
  "list":[]
}
```

用于：

```
收藏页面
```

支持：

- 时间排序
- 分页（后续）

------

## 6. AI生成协议

AI不负责：

- 稀有度
- 概率
- 数据库字段

AI仅负责：

```
文本
图片
```

------

### 6.1 AI文本协议（这里需要注意一下，前端约定的项目规格⭐⭐⭐）

AI返回：

```
Markdown
```

限制：

```
一个一级标题
可使用 ---
可使用 >
（这里需要注意一下，前端约定的项目规格⭐⭐⭐）
不使用HTML
不使用表格
不使用代码块
不使用复杂Markdown
不使用嵌套列表
```

允许：

```
标题
引用
正文
简单强调
```

AI输出应：

- 可解析
- 可阅读
- 不依赖HTML

AI返回示例：

```
# 雨夜与灯火

> 学习并不总是燃烧，也可以是缓慢照亮。

---

窗外下着雨。

有人奔跑，有人停留。

而你仍在向前。
```

------

### 6.2 AI图片协议

图片：

```
512×512
```

作用：

```
卡片主视觉
```

限制：

不得承载：

- 大段文字
- UI组件
- 复杂排版

图片定位：

> 氛围与视觉配图

不承担信息展示。

------

AI返回：

```
{
  "imageURL":""
}
```

URL要求：

```
可访问
可缓存
可保存
```

------

## 7. 稀有度系统（具体规则已经制定，参考「卡片.md」⭐）

稀有度：

由后端计算。

依据：

```
studyDuration
```

AI不参与。

规则：

```
学习时长
↓

概率池

↓

rarity
```

原则：

```
高等级始终稀缺
不保底
不重抽
```

学习记录本身具有价值。

------

## 8. 状态与异常处理

生成过程较长。

前端应支持：

------

生成中：

```
Generating...
```

状态。

------

异常情况：

包括：

1 AI超时

2 图片失败

3 网络失败

4 数据保存失败

后端：

应返回：

```
success=false
```

并包含：

```
message
```

用于提示。

------

建议：

后端生成超时：

```
20~60s
```

内可接受。

----
