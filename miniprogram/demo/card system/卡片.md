# 卡片发布与收集系统

卡片系统作为微信小程序的拓展功能，与主功能保持相对独立，用于根据学习行为生成并收集 AI 卡片。

------

# 功能边界

卡片系统负责：

1. AI生成卡片文本
2. AI生成卡片图片
3. Markdown解析与卡片显示
4. 服务器生成与保存卡片数据
5. 收集与展示页面
6. 稀有度与收集玩法

不负责：

- 学习行为判定逻辑
- 预约与签到系统
- 主业务流程管理

卡片系统仅接收：

> 一次有效学习记录 → 生成卡片

## 字段说明

### 基础字段

```text
uuid
```

卡片唯一ID。

------

```text
userID
```

所属用户ID。

------

```text
cardID
```

卡片模板/生成编号。

用于：

- 调试
- 去重
- 数据分析

------

## 时间字段

### createTime

```text
创建时间
```

生成卡片的服务器时间。

例如：

```text
2026-05-24 19:20:31
```

用于：

- 时间展示
- 排序
- 收藏记录
- 回顾学习历史

------

### studyDuration

```text
本次学习持续时间
```

单位：

```text
分钟
```

例如：

```json
180
```

表示：

> 本次学习180分钟

该字段直接参与：

- 稀有度概率计算
- 展示学习记录

因此：

> 一张3小时白卡也是有效且有意义的数据。

不使用重抽机制。

------

# AI生成数据

AI返回：

```json
{
  "markdown":"",
  "imageURL":""
}
```

服务器负责：

- 稀有度计算
- 主题分配
- 数据落库

AI不决定稀有度。

# AI协议

------

## AI文本 Prompt

AI返回：

```text
Markdown
```

限制：

```text
- 一个一级标题
- 使用 --- 作为卡片分块
- 不使用复杂Markdown结构
- 不使用HTML
- 不使用表格
- 不使用代码块
- 不使用嵌套列表
```

允许：

```text
标题
引用
正文
简单强调
```

------

## Markdown解析规则

------

### 标题

```md
#
```

解析为：

```text
卡片标题
```

------

### 分卡片

```md
---
```

解析为：

```text
卡片内容分块
```

------

### 引用

```md
>
```

解析为：

```text
引用样式
```

用于：

- 名言
- 强调句
- 哲思文本

------

### 正文

普通文本：

```text
正文内容
```

------

## AI图片 Prompt

生成规范：

```text
512×512
```

用于：

- 卡片主视觉图

AI图片仅作为：

> 视觉配图

不得承载：

- 大段文字
- UI元素
- 复杂排版

# 稀有度规则

稀有度由：

```text
学习时长
```

驱动。

原则：

> 时间越长，高品质概率越高
> 顶级卡始终稀缺

不采用：

```text
保底
重抽
必中
```

保留：

> 长时学习仍可能出现低阶卡

学习记录本身即具有价值。

例如：

> 三小时白卡

代表真实投入。

------

## 稀有度等级

| 等级 | 名称 | 边框  | 卡面 |
| ---- | ---- | ----- | ---- |
| N    | 白   | 白/灰 | 普通 |
| R    | 绿   | 绿    | 普通 |
| SR   | 蓝   | 蓝    | 普通 |
| SSR  | 紫   | 紫    | 普通 |
| UR   | 金   | 金    | 特殊 |
| LR   | 红   | 红    | 特殊 |

## 红卡规则

红卡：

```text
LR
```

属于：

> 极稀有资源

规则：

1. 学习时长≥120min
2. 命中概率池

即：

```text
门槛 + 随机
```

红卡不是：

> 高等级卡

而是：

> 事件级收藏

------

# 视觉主题规则

为保证：

- 图片可读性
- 文本阅读体验

颜色不直接影响正文。

采用：

> 边框宣告稀有度

方案。

------

## 主题字段

卡片视觉拆分：

```json
{
  "rarity":"",
  "borderTheme":"",
  "cardTheme":""
}
```

支持未来扩展。

------

## 普通卡

适用：

- 白
- 绿
- 蓝
- 紫

规则：

```text
边框变化
普通卡面
```

不改变整体阅读背景。

------

## 金卡

UR。

允许：

- 金色边框
- 轻主题背景
- 收藏感视觉

避免：

```text
纯黄色
土豪金
```

目标：

> 珍贵感

------

## 红卡

LR。

允许：

- 特殊卡面
- 深色背景
- 纹理
- 特殊视觉

目标：

> 与普通卡形成明显区隔

体现：

> 非普通卡

------

# 主题限制

卡片主题采用：

> 价值观边界 + 主题池

机制。

避免：

- 鸡汤化
- 内容重复
- 单调模板化

------

# 稀有度与主题联动

稀有度不仅影响：

```text
边框
```

同时影响：

```text
内容风格
```

| 稀有度 | 内容特点   |
| ------ | ---------- |
| 白     | 基础鼓励   |
| 绿     | 学习成长   |
| 蓝     | 思考       |
| 紫     | 哲思与审美 |
| 金     | 未来/史诗  |
| 红     | 隐藏主题   |

目标：

形成：

> 收藏欲与未知感

而不仅是颜色差异。

------

**V1 设计原则总结**

```text
学习行为 → 稀有度概率 → 主题池 → AI生成 → 收藏记录
```

卡片系统的核心不是抽卡。

而是：

> 用游戏化收藏，记录真实学习投入。
