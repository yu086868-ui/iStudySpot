## 即梦AI图片生成3.0 Demo 实现要点说明

### 一、API 接口概述

基于火山引擎即梦AI文生图3.0接口，采用**异步任务模式**：
1. **提交任务**：`CVSync2AsyncSubmitTask` - 提交生成请求，返回 `task_id`
2. **查询结果**：`CVSync2AsyncGetResult` - 轮询任务状态，获取生成结果

### 二、火山引擎 V4 签名机制（核心难点）

#### 签名流程
```
1. 构建规范请求
   ↓
2. 构建待签名字符串
   ↓
3. 派生签名密钥
   ↓
4. 计算签名
   ↓
5. 构建 Authorization 请求头
```

#### 关键代码实现

```javascript
// 1. 派生签名密钥
const kDate = HMAC-SHA256(SK, shortDate);        // SK = SecretAccessKey
const kRegion = HMAC-SHA256(kDate, REGION);      // REGION = "cn-north-1"
const kService = HMAC-SHA256(kRegion, SERVICE);  // SERVICE = "cv"
const kSigning = HMAC-SHA256(kService, "request");

// 2. 构建规范请求
CanonicalRequest = [
    HTTPMethod,                    // "POST"
    CanonicalURI,                  // "/"
    CanonicalQueryString,          // "Action=xxx&Version=xxx"
    CanonicalHeaders + "\n",       // "content-type:...\nhost:...\n..."
    SignedHeaders,                 // "content-type;host;x-content-sha256;x-date"
    HexEncode(SHA256(RequestBody)) // 请求体的SHA256哈希
].join("\n");

// 3. 构建待签名字符串
StringToSign = [
    "HMAC-SHA256",
    X-Date,                        // "20250522T123456Z"
    CredentialScope,               // "20250522/cn-north-1/cv/request"
    HexEncode(SHA256(CanonicalRequest))
].join("\n");

// 4. 计算签名
Signature = HexEncode(HMAC-SHA256(kSigning, StringToSign));

// 5. Authorization 请求头格式
Authorization: HMAC-SHA256 Credential={AK}/{CredentialScope}, 
              SignedHeaders={SignedHeaders}, 
              Signature={Signature}
```

#### 必需的请求头
| Header | 说明 |
|--------|------|
| `Content-Type` | `application/json` |
| `Host` | `visual.volcengineapi.com` |
| `X-Date` | UTC时间，格式 `YYYYMMDDTHHMMSSZ` |
| `X-Content-Sha256` | 请求体的SHA256哈希值 |
| `Authorization` | 签名认证信息 |

### 三、接口调用参数

#### 提交任务
```javascript
{
    req_key: "jimeng_t2i_v30",  // 固定值，即梦3.0
    prompt: "描述词",
    seed: -1,                   // -1为随机
    width: 1024,
    height: 1024
}
```

#### 查询结果
```javascript
{
    req_key: "jimeng_t2i_v30",
    task_id: "提交任务返回的ID",
    req_json: JSON.stringify({ return_url: true })  // 返回URL而非base64
}
```

#### 返回结果状态
| status | 说明 |
|--------|------|
| `in_queue` | 任务排队中 |
| `generating` | 生成中 |
| `done` | 完成 |
| `not_found` | 任务未找到 |
| `expired` | 任务已过期 |

### 五、注意事项

1. **AK/SK 安全**：凭证文件不要提交到版本控制
2. **图片有效期**：返回的 `image_urls` 有效期 24 小时，建议及时下载保存
3. **轮询间隔**：建议 3 秒，最多轮询 30 次（约 90 秒超时）
4. **错误码处理**：`code !== 10000` 表示请求失败，需检查 `message` 字段
